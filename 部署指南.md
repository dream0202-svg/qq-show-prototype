# 📚 GitHub Pages 部署指南

## 🎯 快速部署步骤

### 第一步：创建GitHub仓库
1. 访问 [GitHub.com](https://github.com)
2. 点击右上角 "+" → "New repository"
3. 仓库名称建议：`qq-show-prototype`
4. 设置为 **Public** (公开)
5. ✅ 勾选 "Add a README file"
6. 点击 "Create repository"

### 第二步：上传文件
**方法一：网页上传（推荐新手）**
1. 在仓库页面点击 "uploading an existing file"
2. 拖拽以下4个文件到页面：
   - `index.html`
   - `prototype.html` 
   - `qq_character.jpg`
   - `README.md`
3. 在底部填写提交信息：`初始化QQ秀原型项目`
4. 点击 "Commit changes"

**方法二：Git命令行**
```bash
git clone https://github.com/你的用户名/qq-show-prototype.git
cd qq-show-prototype
# 复制所有文件到这个目录
git add .
git commit -m "初始化QQ秀原型项目"
git push origin main
```

### 第三步：启用GitHub Pages
1. 在仓库页面点击 "Settings" 标签
2. 在左侧菜单找到 "Pages"
3. 在 "Source" 部分选择：
   - Source: **Deploy from a branch**
   - Branch: **main** (或 master)
   - Folder: **/ (root)**
4. 点击 "Save"
5. 等待1-2分钟，页面会显示访问链接

### 第四步：获取访问链接
✅ **您的原型地址将是**：
```
https://你的GitHub用户名.github.io/qq-show-prototype
```

## 📱 手机访问方式

## 🔧 常见问题解决

### Q: 页面显示404错误
**A**: 检查以下几点：
- 确保仓库是Public（公开）
- 确保已启用GitHub Pages
- 等待5-10分钟让部署生效
- 检查文件名是否正确（index.html）

### Q: 图片不显示
**A**: 确保 `qq_character.jpg` 文件已上传到仓库根目录

### Q: 手机上交互不流畅
**A**: 
- 使用现代浏览器（Safari、Chrome）
- 确保网络连接稳定
- 清除浏览器缓存后重试

### Q: 想要自定义域名
**A**: 
1. 在仓库根目录创建 `CNAME` 文件
2. 文件内容写入您的域名（如：qq-show.yourdomain.com）
3. 在域名DNS设置中添加CNAME记录指向 `你的用户名.github.io`
